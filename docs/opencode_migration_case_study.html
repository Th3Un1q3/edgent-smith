Case Study

Migrating to a
Local Agentic CLI

Dropping Github Copilot after it burned all premium credits in 3 hours for an offline, isolated OpenCode environment.

One Does Not Simply Point Copilot to a Local Model

If you point GitHub Copilot directly to a locally running LLM, the system breaks instantly. Any model you can run locally is simply too slow and too dumb to process the massive, unoptimized context payloads generated by Copilot Chat. To make local setups work, you must switch from working hard to working smart.

Context Flooding

A single normal query in GitHub Copilot Chat immediately dumps 40,000+ tokens of raw codebase data. Any locally hosted model running on consumer hardware will choke, freeze, or crash under this payload.

Lack of Context

Proprietary frontier models provided by the labs Are quite capable. They're good at guessing things right and don't give up on simple errors. Local models are different.

Four Pillars of the System

An architectural breakdown of the key elements supporting a fast, reliable, and private local development ecosystem.

1. Local LLM and Backend

Requirement: Fast, performant, and efficient model running.

Failed Paths & Dead Ends
  • Qwen 3.5 27B (Ollama): Too large, generating only 5 tokens/sec.
  • Ollama as model runner: Update Cadence too slow for cutting-edge models.
  • Gemma 4 4B / Small Models: Lazy, failed complex logic.
Selected Stack

LM Studio + Gemma 4 12B (Quantization Aware Training): Provides hardware acceleration for faster generation and makes small model feel as capable as larger models.

Next-Gen Upgrade

The next-generation improvement lies in either next-release models or substantial hardware advancements.

2. System Resources

Requirement: Sufficient resources to keep the host machine performant and responsive.

Failed Paths & Dead Ends
  • Docker Desktop (Default Virtualization): Consumed 24 GB of RAM unnecessarily, starving the local LLM.
Selected Stack

Apple Virtualization Framework: Native reclamation of precious memory for side-by-side performance.

Next-Gen Upgrade

Replacing Docker Desktop with OrbStack achieves superior resource efficiency and allows for continuous memory footprint reduction.

3. Agentic System

Requirement: An agentic system that integrates seamlessly with locally running models.

Failed Paths & Dead Ends
  • oh-my-opencode (Ultrawork-style): The plugin demanded extremely high token throughput, which overwhelmed local models constrained by limited generation capacity.
  • Direct OpenCode Execution: Small models are too lazy, look in wrong places, and give up immediately upon encountering a first error instead of troubleshooting workloads.
  • VS Code + Github Copilot (Local Model): Attempting to point Github Copilot to a local model was too slow and inefficient; its internal limitations (tool exposure, context systems, and journals) made performance improvements impossible.
Selected Stack

Opencode + OpenAgentsControl: Agentic design with incremental context discovery and robust workflow management.

Next-Gen Upgrade

Future improvements include exploring advanced context and memory management techniques, seeking enhancing plugins, and implementing mixed-model approaches for optimal task-specific performance.

4. Tooling Gateway

Requirement: The most efficient, extensible, and portable method for interacting with tools.

Failed Paths & Dead Ends
  • Stock VS Code + Copilot Extension: Uses a proprietary MCP connection setup, making it difficult to share across multiple agents; migrating for cross-agent support requires significant effort without providing performance gains.
  • Router Nexus Gateway: A two-tool approach (search and execute) requires excessive agent labor to understand, is unreliable, and fails to provide significant performance bonuses. It still limits execution to one tool at a time, lacks return value preprocessing, and remains susceptible to context flooding from MCP responses.
Selected Stack

Docker MCP Gateway (Code Mode): Enables progressive tool discovery and effective multi-tool chaining. Allows agents to execute multiple tools in a single run, truncate responses to prevent context flooding, and focus only on relevant results, making complex coding tasks significantly easier.

Next-Gen Upgrade

Scaling with more MCPs: The architecture supports continuous expansion by adding more MCP instances and training agents to interact with them effectively.
